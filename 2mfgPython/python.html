Emotion Based Movie Recommendation App

January 2023 - April 2023

Arizona State University, Tempe, Arizona

  • This project can be accessed through this Google Drive link: Google Drive Link
  • Please refer to this video link for more information on the project: Project Video Link
  • Before proceeding, download all the files from the drive.
  • Run the MFG_Project_Main_Code.py file and refer to the video link below for guidance before running the code.
  • The MFG_Project_Main_Code.py file runs two Python files as applications.
  • Upon running the file, the MFG598_Emotion_Detection.py code opens as app1, which captures an image of your current emotional state and saves it as a photo.png file.
  • The saved photo is then used to predict the emotion using the ferNet.h5 model, which uses KNN to predict emotion by comparing the facial photos in the train folder.
  • The ferNet.h5 model is a well-known model trained from FER2013 (Facial Expression Recognition 2013 Dataset).
  • The FER2013 contains approximately 30,000 facial RGB images of different expressions, with a size restricted to 48Ã—48 pixels, and the primary labels of the dataset can be divided into seven types: 0=Angry, 1=Disgust, 2=Fear, 3=Happy, 4=Sad, 5=Surprise, 6=Neutral.
  • The predicted emotion is saved in the emotion_out.txt file.
  • Using the saved emotion_out.txt file, the second code, MFG598_Movie_Recommend_Emotion_Based.py, is opened as app2.
  • Seven CSV files have been created for movie recommendations based on specific emotions, namely anger, disgust, fear, happiness, neutral, sadness, and surprise. These files contain movie names, genres, and ratings for the respective emotions, and are based on research papers.
  • Based on the saved text file, the app will open the respective emotion.csv file and display the movie recommendation for the current emotion. If the user doesn't like the recommended movie, they can click the next button, which will recommend another movie a thousand times.
  • If the user likes the movie, they can press the movie button, which saves the movie name as selected_movie.txt.
  • In the future, if you wish to watch a movie similar to the recommended one, I have created another app that recommends movies based on the plot and content of the recommended movie, using the tmdb_5000_credits.csv and tmdb_5000_movies.csv files.
  • In the MFG598_Movie_Recommend_Content_Based.py app, the code first prompts the user to enter the movie name (case-sensitive). Based on the information provided, the app uses a TfidfVectorizer to convert the movie overviews into vectors and calculates the cosine similarity between them.
  • It then creates a mapping between movie titles and their corresponding indices in the dataset and finds the movies with the highest cosine similarity scores to the input movie title.
  • The app displays the top 10 recommended movies and allows the user to load more recommendations by clicking a button.
  • If the input movie name is not available in tmdb_5000_movies.csv, the app will prompt the user to input a different movie.
  • To close all the applications, press the Esc key.

    • Conclusion

    Our project combines emotion detection and movie recommendations to offer a unique and personalized entertainment experience. Through facial expression analysis using the FER2013 dataset and ferNet.h5 model, we accurately predict emotions and recommend movies that resonate with the user's feelings. Additionally, our content-based movie recommendation system utilizes natural language processing to suggest films based on plot and content, providing a holistic approach to movie recommendations. Our project showcases the power of machine learning and data-driven solutions in enhancing everyday entertainment choices, aiming to offer users an immersive and tailored movie-watching experience.
